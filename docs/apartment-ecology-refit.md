# 公寓生态缝合说明（消化住户 × 校源辅锚）

## 1. 公寓 NPC 新分层

| 层级 | 含义 | 代表 |
|------|------|------|
| **消化链住户** | 公寓泡层碎片、旧住户、污染残留中形成的 NPC；动机多与「被消化/抵抗消化」绑定 | 陈婆婆、林医生、邮差、阿花、盲人、电工、洗衣房阿姨、厨师、保洁员等 |
| **校源耦合辅锚（六人）** | 耶里事故卷入者，被循环改写成公寓职能壳；**与消化住户共用电梯、规则、服务节点** | 麟泽、灵伤、欣蓝、北夏、枫、叶 |
| **秩序/账簿节点** | 消化系统的「管理叙述层」，与登记口、交换链存在张力但非简单上下级 | 夜读老人（N-011） |
| **高危高层变量** | 7F 物理消化前线 + 话术诱导叠层 | 厨师、枫同层动线、前调查员等 |
| **失败轮次残留** | 由循环/纠错窗口产生的残响（多由任务与 reveal 门闸表现，不单开 id） | 主锚复活节拍、六人 residual_echo |

六人 **不是** 另一张地图的队友，而是 **同一栋楼里多一类「岗位型徘徊者」**：表层被住户当成物业/补给/商人/画室/7F 前线，深层才接校源线。

## 2. 六人与普通 NPC 的关系模型（设计原则）

- **老徘徊者视角**：陈婆婆、老刘、洗衣房阿姨等把辅锚当「太像执勤表的人」「制服太新的人」，**不**必知耶里校名。
- **知情/半知情**：林医生、邮差、前调查员等把登记、交换、701 话术当 **行政或病理样本**。
- **危险变量视角**：阿花、失眠者、厨师等从 **恐惧或频段冲突** 感知 7F 与话术，不必懂七锚。
- **共事**：B1 后勤链上灵伤、麟泽与 N-008/N-014 **并发**出现；服务节点 `npcIds` 显式包含辅锚与普通后勤，避免「商店只属于背景 NPC」。
- **误解与偏见**：老刘对登记口径、陈婆婆对灵伤的笑——**张力在职能与账簿，不在「主角团特权」**。

关系边已写入 **`NPC_SOCIAL_GRAPH`**（`world.ts`）中普通 NPC 节点；六人正文仍以 **`majorNpcDeepCanon` + patch** 为准。

## 3. 服务节点为何仍合理

- **B1** 是公寓 **绝对安全区 + 系统齿轮枢纽**；辅锚之二（麟泽、灵伤）常驻 B1，出现在 **复活锚点、商店、谣言、柔性引导** 的 `npcIds` 中，表示 **同节点可有多张脸**：电工修灯、阿姨洗单、灵伤对补给、麟泽对边界见证。
- **锻造/薪资** 仍以 N-008 为主力（契约稳定）；**未**改 service id。
- 高层 **7F** 仍以厨师、调查员、枫的动线并存：**物理消化** 与 **话术诱导** 同层争频段（厨师 lore / 社交图已点明）。

## 4. 玩家为何不应感到割裂

1. **同一物理规则**：都要登记、上楼、遇威胁、用 B1 服务；辅锚受 **relink / reveal / 任务** 门闸，不默认跟队。  
2. **普通 NPC 有主线服务价值**：误导、传言、见证、旁证、恐惧、偏见、历史残留——已写入多人民间 lore 与关系边（见 `npcs.ts` + `world.ts`）。  
3. **世界知识种子**：`registryAdapters` 为每个 NPC 增加 **`ecology:*` 标签** 与 **【生态缝合】** chunk，检索时可与 `digestion_resident` / `yeliri_coupled` 同现，避免 RAG 只抽到「孤立主角团」。  

## 5. 旧设定吸收 vs 移除

| 区域 | 吸收方式 | 移除/占位方式 |
|------|----------|----------------|
| 六人旧职（电梯工/诱饵/无面等） | 吸收为 **公寓职能壳**；玩法上镜面委托、补给、登记仍可用 | **静态 `world.ts`** 中 N-007/N-013/N-015/N-018/N-020 大块旧叙事改为 **patch 占位**，防止误维护 |
| 北夏与镜像/倒行者 | 保留 **任务与镜面** 作交换壳，与校源外联身份并存 | 不再把 **无面保安** 写进静态 immutable |
| 灵伤与陈婆婆 | 保留 **不信任+仍接补给** 的民间张力 | 移除「直属物业经理执行指令」类 **科层** 绑定（静态块已不占优；以 deep canon + patch 为准） |
| 物品/仓库 | 此前已改文案专名与 **ownerId 不变** | 无新增 id 变更 |

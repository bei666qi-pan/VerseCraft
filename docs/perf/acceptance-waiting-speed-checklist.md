## 等待/速度优化验收清单（可执行，≥20 条）

格式：**场景 / 操作 / 预期结果 / 不应发生的问题**

---

1) **普通短输入（快车道）**
- **操作**：进入 `/play`，输入一句短行动（≤20字），发送 5 次。
- **预期结果**：2 秒内出现等待文案推进，10 秒内大概率进入正文流；`chat_client_perf.firstPerceivedFeedbackMs` 明显低。
- **不应发生**：等待态长时间停在“生成中”；卡死无反馈；自动补选项反复触发。

2) **普通长输入（慢车道倾向）**
- **操作**：发送一段多分句长行动（≥120字）。
- **预期结果**：等待阶段更靠后（context/build/generating），但推进稳定不闪烁；最终正文质量不明显劣化。
- **不应发生**：频繁跳阶段、回退、抖动；过早显示“正文流动中”但长时间无字。

3) **慢车道（安全/复杂指令）**
- **操作**：发送疑似复杂系统指令/多条件行动。
- **预期结果**：仍通过输入安全与必要风控；`riskLane=slow` 的 TTFT 合理但不无上限。
- **不应发生**：绕过安全链；玩法状态被破坏。

4) **开场正常**
- **操作**：新会话首次进入游戏，等待开场完成。
- **预期结果**：开场正文与 UI 不假死；若选项缺失，会进入低成本 options-only 补齐。
- **不应发生**：开场反复重试主链路导致等待累加到分钟级。

5) **开场上游慢/断（强制）**
- **操作**：模拟上游故障（断网/网关错误），进入开场。
- **预期结果**：14 秒左右进入“开局无法获取选项”的兜底提示，或出现 options-only 兜底；UI 可操作（可切手动输入）。
- **不应发生**：一直转圈不收敛；不断重发开场主链路。

6) **options-only 手动刷新**
- **操作**：点击“刷新选项”按钮。
- **预期结果**：进入轻量提示；在 9–12 秒内成功则更新 options；失败则快速回到可操作状态并提示切换手动输入。
- **不应发生**：补选项把用户锁住很久；补选项触发主链路长等待。

7) **auto_missing_main（正文有但 options 缺失）**
- **操作**：构造一次让模型返回无 options 的回合（或模拟）。
- **预期结果**：自动触发一次低成本补选项；预算命中则给保守兜底，不拖长总等待。
- **不应发生**：多次自动补救循环；总等待显著变长。

8) **mode switch text→options（用户主动）**
- **操作**：切换到 text 再切回 options（且无 options）。
- **预期结果**：最多触发一次自动补选项，30s 内不重复；用户可继续操作。
- **不应发生**：每次切换都触发额外等待。

9) **mode switch 仅切换不需要补**
- **操作**：有 options 时反复切换 text/options。
- **预期结果**：不发起 options-only 请求。
- **不应发生**：无意义补选项请求。

10) **决策回合 decision fallback**
- **操作**：构造 decision_required 但 decision_options 不足的回合。
- **预期结果**：补决策选项预算严格（≤2.2s/1.8s）；失败也能给保守兜底。
- **不应发生**：补决策导致回合等待显著延长。

11) **流式首字节/headers 很慢**
- **操作**：模拟服务端极慢响应头（本机压测/限速）。
- **预期结果**：在 `FETCH_CHAT_RESPONSE_DEADLINE_MS` 内有明确提示；不会无限等。
- **不应发生**：长时间空白无反馈。

12) **首 chunk stall**
- **操作**：让服务端回 headers 但不写 `data:`（模拟代理缓冲）。
- **预期结果**：`STREAM_FIRST_CHUNK_STALL_MS` 到点收敛提示。
- **不应发生**：一直“正在生成…”。

13) **中途 chunk 长停顿**
- **操作**：模拟 chunk 间隔>70s。
- **预期结果**：触发 stall 收敛提示；不把用户锁死。
- **不应发生**：无期限等待。

14) **reconnect：EMPTY_CONTENT**
- **操作**：模拟上游返回极短/空输出。
- **预期结果**：最多 1 次同类 reconnect，最多 2 轮总尝试；超过 40s 直接 fallback。
- **不应发生**：反复 reconnect 叠加到分钟级。

15) **reconnect：STREAM_INTERRUPTED**
- **操作**：模拟中途断流。
- **预期结果**：最多 1 次同类 reconnect，失败则 fallback。
- **不应发生**：多轮重连拖长等待。

16) **finalizing 过长**
- **操作**：模拟后处理很慢（输出审核/终帧处理）。
- **预期结果**：等待阶段能合理推进到 finalizing；最终仍能收敛（成功或降级）。
- **不应发生**：正文已出但 UI 长时间挂起。

17) **游客模式对话次数耗尽**
- **操作**：游客达到 50 次对话限制后继续发送。
- **预期结果**：出现 paywall/提示，不触发额外补救请求。
- **不应发生**：继续发起模型请求或补救链。

18) **存档恢复继续执笔**
- **操作**：刷新页面后进入 `/play`，触发 resume 逻辑。
- **预期结果**：恢复请求稳定；不重复触发开场与补选项；等待态不悬挂。
- **不应发生**：重复 sendAction、重复补选项、旧 options 回填。

19) **终局**
- **操作**：触发终局回合并推进。
- **预期结果**：终局唯一选项不被补选项逻辑干扰；等待态正常。
- **不应发生**：options-only 补救介入终局；多余重试。

20) **结算跳转**
- **操作**：触发死亡/理智归零并等待跳转。
- **预期结果**：`tail_draining` 后 2s 跳转结算；等待态不悬挂。
- **不应发生**：卡在 waiting_upstream/streaming_body。

21) **上游 401/403**
- **操作**：配置错误密钥，发送请求。
- **预期结果**：更快失败并降级提示；不长时间切模型。
- **不应发生**：长链式重试把等待拖很长。

22) **上游 429**
- **操作**：模拟限流（429）。
- **预期结果**：更快失败并提示稍后重试；不长时间切模型。
- **不应发生**：长链式 fallback 重试。

# 当前仍需运营 / 法务 / 产品配合的事项清单

以下事项**无法**仅靠仓库代码关闭，需组织内流程与外部专业支持。

## 运营与客服

- [ ] **电话线路**：公示号码的接听时段、话术、与在线表单（VC-COMP-）的工单对应关系。
- [ ] **举报与申诉 SLA**：内部目标时限、升级路径、与「不承诺固定完成时点」页面对用户的预期管理。
- [ ] **内容审核**：违法违规、侵权、诈骗等类别的实人复核流程与记录保存策略。

## 数据主体权利（删除 / 导出 / 更正）

- [ ] **身份核验 SOP**：何种账号信息算「可核验」，拒绝请求的条件与留痕。
- [ ] **执行路径**：数据库删改、对象存储、备份与日志留存是否一并处理；代码仅提供受理入口，**不替代**运维执行表。
- [ ] **第三方**：若经 one-api / 云厂商处理，对外说明与分包合同是否与隐私政策一致。

## 法务与资质

- [ ] **法律顾问审阅**：用户协议、隐私政策、未成年人条款、AI 披露在**当前商业模式**下的定稿。
- [ ] **产品分类与资质**：是否适用网络游戏、生成式 AI 备案/标注等——需专项结论，代码与文档**不给出**监管定性。
- [ ] **主体信息**：营业执照名称与 `NEXT_PUBLIC_OPERATING_SUBJECT`、ICP 主体一致。

## 技术与安全（跨团队）

- [ ] **生产迁移**：确认 `compliance_inquiries` 已在目标库创建。
- [ ] **监控与导出**：受理表是否纳入备份与访问审计（仅技术方案，合规意义由法务评估）。

在 `docs/open-beta-compliance-upgrade.md` 中交叉引用重大状态变化。

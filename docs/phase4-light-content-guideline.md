# Phase 4 轻内容扩展规范（防膨胀）

## 目标

Phase 4 只扩展“内容密度与路线表达”，不扩展重系统体量。  
优先保证 Stage 3 的稳定性与可复用性。

## 允许做的事

- 在现有任务框架内补短任务、分支后续任务、关系分流任务。
- 在现有 codex/truth 结构内补线索层次与证据链。
- 在现有 threat/weapon/forge 包上做轻量策略提示优化。
- 在现有职业收益框架内做小幅参数校准（冷却、提示语、触发阈值）。

## 明确不做

- 大技能树 / 天赋盘
- 混职 / 双职业 / 职业组合技
- 分支合并 / 跨分支继承经济
- 新的重型经济系统
- 重型职业专属 UI 页面
- 直接跳过 threat 核心压力的“职业无敌解”

## 设计边界

1. 属性是底座，职业是放大器，不可反客为主。  
2. 所有职业收益必须落在已有业务字段回写上（task/threat/weapon/codex/relationship）。  
3. narrative 不能替代状态写回。  
4. 每次新增内容必须可在分支隔离下独立回放。  
5. 新内容默认“小步快跑”：单批最多 1-2 个玩法点。

## 变更清单模板（每次 PR 必填）

- 本次新增内容点：
- 影响系统（threat/weapon/forge/task/codex/relationship）：
- 是否新增持久化字段：
- 是否新增 packet 字段：
- 分支隔离影响评估：
- 回归测试列表：
- 明确未做项：

## 验收红线

- 不允许出现 narrative 转职但 state 未写入。
- 不允许分支 UI 有变化但快照无变化。
- 不允许跨分支共享职业认证与奖励结果。
- 不允许职业收益成为主数值系统。
